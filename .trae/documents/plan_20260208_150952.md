# 修复中文显示问题 - 编码不匹配

## 问题根源
源代码使用 UTF-8 编码，但字模表使用 GBK 编码，导致匹配失败。

## 解决方案（方案1 - 推荐）
修改 main.c 中的中文显示调用，直接传入 GBK 编码的字节数组：

```c
// 原代码（UTF-8 编码，无法匹配）
lcd_show_chinese(10, 50, (uint8_t*)"遇见司空", RED, WHITE);

// 修改为（GBK 编码字节数组）
uint8_t chinese_str[] = {0xD3, 0xF6, 0xBC, 0xFB, 0xCB, 0xA9, 0xBF, 0xD5, 0x00};  // 遇见司空
lcd_show_chinese(10, 50, chinese_str, RED, WHITE);
```

## 修改文件
- `Core/Src/main.c`：修改中文显示调用方式

## 说明
- 0xD3 0xF6 = 遇（GBK）
- 0xBC 0xFB = 见（GBK）
- 0xCB 0xA9 = 司（GBK）
- 0xBF 0xD5 = 空（GBK）
- 0x00 = 字符串结束符
